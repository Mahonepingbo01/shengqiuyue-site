# Security Policy

## Reporting a Vulnerability

If you find a security issue, please open a GitHub issue with a clear description.

Please do not include real secrets, API keys, passwords, or private credentials in public issues.

Useful reports include:

- exposed secrets
- unsafe configuration examples
- vulnerable dependencies
- insecure automation scripts
- unsafe API usage patterns
- permission problems
- deployment risks

## Security Goals

This project aims to keep examples safe for creators, solo builders, and small teams who may adapt this repository for their own AI-assisted websites and workflows.
